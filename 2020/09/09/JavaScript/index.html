原型

对象默认的 __proto__ 属性所指向的就是该对象的原型，原型也是一个对象。

规则

  • 对象分为普通对象和函数对象。
  • 所有对象默认有一个 __proto__ 属性。
  • 只有函数对象默认有一个 prototype 属性。
  • Function 和 Object 是默认的函数对象，都是通过 new Function 实例出来的对象。

instanceof

instanceof 运算符的第一个变量是一个对象，暂时称为A；第二个变量一般是一个函数，暂时称为B。
instanceof 的判断规则是：沿着A的 __proto__ 这条线来找，同时沿着B的 prototype 这条线来找，如果两条线能找到同一个原型对象，那么就返回true。如果找到终点还未重合，则返回false。

所以 instanceof 的作用就是判断一个实例是否是其父类型或祖先类型的实例。

举例思考

例 1

  1. f1.prototype 是？
  2. f1.__proto__ 是？
  3. Foo.__proto__ 是？
  4. Foo.prototype.__proto__ 是？
  5. Function.__proto__ 是？
  6. Object.__proto__ 是？
  7. Function.prototype.__proto__ 是？
  8. Object.prototype.__proto__ 是？

答案：

  1. undefined。普通对象中不存在 prototype 属性。
  2. Foo.prototype。f1 的构造函数是 Foo。
  3. Function.prototype。Foo 的构造函数是 Function。
  4. Object.prototype。Foo.prototype 的构造函数是 Object。
  5. Function.prototype。Function 的构造函数是 Function。
  6. Function.prototype。Object 的构造函数是 Function。
  7. Object.prototype，Function.prototype 的构造函数是 Object。
  8. null。Object.prototype 在原型链顶层。

例 2

  1. Function instanceof Function 是？
  2. Object instanceof Function 是？
  3. Function instanceof Object 是？
  4. Object instanceof Object 是？

答案：

  1. true。参照 instanceof 的判断规则。
  2. true。
